## 一、最相似项目（技术栈重合度高）

### 1. ⭐ xxanvv/Video-Downloader — 架构最像你

| 项目 | 详情 |
|------|------|
| **GitHub** | [xxanvv/Video-Downloader](https://github.com/xxanvv/Video-Downloader) |
| **技术栈** | Python + **PyQt6** + yt-dlp |
| **特点** | 现代化暗色主题、并发多任务下载、暂停/恢复、实时速度/进度/ETA、批量 URL 处理 |

**Origami 对比：**
- ✅ 相同栈（PyQt6）
- ✅ 并发下载
- ❌ 基于 yt-dlp（依赖社区维护 extractor），Origami 自己写 platform adapter
- ❌ 没有「登录→下载」的完整闭环，Origami 有 WebView 登录 + Cookie 管理
- ❌ 没有托盘最小化、单实例检测

**Origami 优势：** 反爬更强（Puppeteer + sign-server）、UI 更精致（暗色/亮色主题切换）、真多平台架构（platform adapter 插件式）

**核心功能：**

2. **商业化背书** — TikHub 是正规商业服务，有法律团队
3. **Apache 2.0 许可证** — 比 MIT 多一层专利/责任保护
4. **多平台** — 不单绑抖音，支持 TikTok + B站
5. **docker.wtf 演示站明确关闭下载** — 演示只展示能力，不下场做下载

## 四、Origami 的差异化优势

1. **Puppeteer 智能反爬** — 独家，所有竞品要么依赖 yt-dlp、要么手动填 Cookie
2. **插件式平台架构** — `PlatformAdapter` 抽象层，新平台三步入驻，架构最优雅
3. **sign-server** — 独立 Node.js 签名服务，绕过了 yt-dlp 的抖音 extractor 不稳定问题
4. **完整桌面体验** — 托盘 + 单实例 + 快捷键 + 主题切换，竞品大多缺胳膊少腿
5. **代码签名** — SignPath 免费签名，不会被 Windows Defender 误杀
6. **Inno Setup 安装器** — 专业级安装体验，竞品基本都是绿色版

## 五½、重点深挖：Evil0ctal 可借鉴清单

> 这是同类项目中**代码质量最高、架构最成熟、且仍在存活**的项目。
> 逐文件分析后，整理以下可落地借鉴点。

### 架构层面

| 借鉴点 | Evil0ctal 的实现 | Origami 现状 | 怎么做 |
|--------|-----------------|-------------|--------|
| **异步 HTTP 客户端** | `httpx.AsyncClient` + `asyncio.Semaphore` 控制并发 | `requests` 同步阻塞 | 将 `DeepSeekService` 级别的 HTTP 换成 httpx 异步 |
| **平台子目录结构** | `crawlers/douyin/web/` 含 `models.py` + `endpoints.py` + `config.yaml` + `utils.py` | 所有平台适配器平铺在 `src/platforms/` | 改成 `platforms/douyin/` 目录结构，按文件拆分 |
| **配置驱动** | 每个平台有独立 `config.yaml`（Headers、Cookie、域名、msToken等） | 硬编码在 adapter 里 | 新增 `config.yaml` 每个平台一个，敏感信息不入库 |
| **Hybrid 聚合层** | `hybrid_crawler.py` 统一调度多平台，自动识别 URL 路由 | 无 | 新增 `src/platforms/router.py`：输入 URL → 自动匹配平台 |
| **Pydantic 数据模型** | 请求/响应都用 `pydantic.BaseModel` 严格校验 | 用 `@dataclass`，无校验 | 引入 pydantic 做响应校验，防止 API 变更导致崩溃 |

### 工程层面

| 借鉴点 | Evil0ctal 的实现 | Origami 现状 | 怎么做 |
|--------|-----------------|-------------|--------|
| **异常体系** | 7 种 `APIError` 子类：Connection/Response/NotFound/RateLimit/Timeout/Unauthorized/RetryExhausted | 只有一个通用 try-except | 新增 `src/exceptions.py`，每种错误有独立处理和用户提示 |
| **重试机制** | BaseCrawler 内置 3 次重试 + `asyncio.sleep(timeout)` 退避 | 无重试 | BaseCrawler 层加 retry 逻辑 |
| **Docker 部署** | `Dockerfile` + `docker-compose.yml` | 仅 Windows exe | 未来做 Web 版时可参考；桌面端暂不需要 |
| **PyPI 发布** | `douyin-tiktok-scraper` pip 包，可独立使用 | 无 | 把 crawler 核心抽成 `origami-core` 包发布到 PyPI（简历加分巨大） |
| **API 文档化** | FastAPI 自动生成 Swagger UI | 无 | 不做 Web 服务的话不需要 |

### 可以直接抄的

| 文件/模块 | 路径 | 说明 |
|----------|------|------|
| **错误类型定义** | `crawlers/utils/api_exceptions.py` | 7 种 APIError 子类，直接翻译成 Origami 风格 |
| **抖音端点全集** | `crawlers/douyin/web/endpoints.py` | 153 行，列出了抖音所有已知 API 端点——比 Origami 现有的丰富得多 |
| **抖音请求模型** | `crawlers/douyin/web/models.py` | 用 Pydantic 定义了 30+ 请求模型（UserPost/PostDetail/UserLike...），可以直接参考字段 |
| **B站 adapter 骨架** | `crawlers/bilibili/web/` | 完整的 B站实现，Origami 做 B站 adapter 时可以直接参考 |
| **URL 路由逻辑** | `hybrid/hybrid_crawler.py` | 混合爬虫如何自动识别链接属于哪个平台，Origami 的 `resolve_url` 可以升级 |
| **Bogus 算法** | `crawlers/douyin/web/xbogus.py` + `abogus.py` | 如果未来想摆脱 Node.js sign-server 依赖，可以移植 Python 实现 |
| **配置模板** | `crawlers/douyin/web/config.yaml` | 配置驱动的设计，每个平台一个 yaml，敏感信息（Cookie）不入 git |

### 优先级

```
立即抄: 异常体系 + 端点全集（补充 Origami 缺的 API）
B站前抄: 参考 bilibili/ 目录结构，搭 B站 adapter 骨架
中期抄: 异步 httpx + Pydantic 校验 + config.yaml 配置驱动
长期抄: PyPI 发布 + URL 自动路由 + Bogus Python 化
```

## 六、总结

> **Origami 在同类项目中属于「技术栈最新、架构最优雅、反爬最强」的梯队**，但功能广度和社区规模不如 erma0/douyin (1.6k stars, 7年积累) 和 Evil0ctal/Douyin_TikTok_Download_API（同类最高 Star）。你的核心壁垒是 Puppeteer + sign-server 的反爬体系 + 插件式多平台架构，这是任何一个竞品都没有的。

### 两个关键对照

| | erma0（已死） | Evil0ctal（存活） | Origami |
|------|-----------|-----------------|---------|
| 定位 | 抖音爬虫 | 开发者 API / 开源库 | 多平台内容管理工具 |
| 形态 | Web + GUI + CLI | Web API + Web UI | **纯本地桌面端** |
| 商业化 | 无 | TikHub API 服务 | 无 |
| 多平台 | ❌ | ✅ 抖音+TikTok+B站 | ✅ 架构就绪 |
| 命运 | 2026.6 清空 | 活得好好的 | — |

**Evil0ctal 为什么比 erma0 安全？**

不是技术更好——而是**定位 + 多平台 + 商业化**三层保护：
1. 面向开发者（非终端用户），不宣传「免费下载」
2. 已支持 3 个平台，抖音只是之一
3. 商业化 API（TikHub）有法律实体和商业合同做护盾
4. 演示站明示「关闭下载功能」

**Origami 的路：**
短中期学 Evil0ctal 的多平台（B站 adapter 是最高优先级），长期保持桌面端本地优势。纯本地桌面端**天然比 Web 服务安全**——你不提供服务，就没有「提供侵权服务」的法律责任。这是 Origami 最核心的合规护身符。

- 数据采集：用户主页、作品详情、评论区（多级回复）、搜索、关注/粉丝列表、收藏、推荐流

- ❌ Origami 无 proxy 代理支持
